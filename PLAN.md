# Kernel Fuzzing Plan

Extend lean-fuzz from surface-syntax fuzzing into Lean 4 kernel fuzzing via differential testing across multiple kernel implementations, driven by AFL++ for coverage-guided mutation.

## Targets
Kernel crashes (SIGSEGV, ASAN, UBSAN), assertion failures, hangs/timeouts, and verdict disagreements between kernels. Any of these is a finding. Constructing `False` is *not* the primary goal â€” it's exceedingly rare; crashes and disagreements are the realistic bug class.

## Oracle: Differential Crash
Run each generated input through every available kernel. If verdicts disagree, the harness deliberately aborts (`IO.Process.exit 139`), which AFL records as a crash and writes to `findings/`. Native crashes and timeouts in any kernel are detected by AFL natively (exit signal / wall-clock).

## Kernels
- **C++ kernel** â€” Lean's reference, in-process via `Environment.addDecl` and the kernel-check path
- **lean4lean** (Mario Carneiro) â€” Lean reimplementation, added as a Lake dep, second in-process verdict
- **nanoda** â€” minimal Rust kernel, out-of-process via `lean4export` text format

Each generator path tags itself with which kernel feature subsets it uses. Per-session runs filter to the intersection of supported features so unsupported-feature rejections aren't mistaken for disagreements.

## Generator Pyramid
- **`GenRawExprM`** â€” untyped `Expr` emission. Bad bvar indices, exotic universe levels, malformed `proj`, deep nesting. Kernel is supposed to reject these cleanly; finding paths that don't is the bug.
- **`GenExprM`** â€” forward type-directed composition in `MetaM`. Maintains a typed term pool from env constants + locals, applies functions to compatible arguments, type emerges bottom-up. Never tries to inhabit an arbitrary goal type (that's proof search).
- **`GenDeclM`** â€” random `Declaration`s, especially inductive types. Highest historical kernel-bug density (positivity, recursor synthesis, mutual/nested inductives).
- **Tactic-as-generator** â€” only goal-driven path. Generate a goal Prop, dispatch to `Canonical` / `aesop` / `exact?` / `omega` / `decide` / `simp` with timeout, kernel-check whatever term they synthesize. Tactics produce term shapes structural generators never reach.

The existing `GenM` (surface syntax â†’ string) remains and gets retrofitted to the same byte-stream RNG; it feeds the parser+elaborator+kernel pipeline as a fourth path.

## Coverage-Guided Driver: AFL++
Compile via `LEAN_CC=afl-clang-fast LEAN_CXX=afl-clang-fast++ lake build`. This instruments all C generated by Lean â€” covers the harness, the generator, and lean4lean (which is in Lean and compiles through `LEAN_CC`). The C++ kernel and Lean runtime stay uninstrumented at first because they're prebuilt in the toolchain; crashes in them are still detected (AFL sees exit signals regardless), just unsteered. Rebuilding the toolchain with `afl-clang-fast` is a future optimization, not blocking.

Build nanoda with `cargo-afl` for Rust-side instrumentation.

**Run mode: AFL parallel.** One AFL instance against the Lean harness binary, one against nanoda, both pointing at the same `-o` findings dir. AFL automatically cross-pollinates corpora between instances. Simpler than shared-memory coverage merge across subprocesses; same bugs found.

If parallel mode proves limiting, the next step is shared `__AFL_SHM_ID`: parent Lean harness spawns nanoda as a subprocess inheriting the env var, both write to the same coverage map.

## Harness Shape
Single Lean binary. `main` reads bytes from stdin or argv:
1. One-time init: load oleans, set up `Environment`, prepare lean4lean
2. Per iteration: bytes â†’ generator's choice oracle â†’ produce `Expr` / `Declaration` / syntax string
3. Run input through C++ kernel and lean4lean in-process
4. (Optional, if not in AFL parallel mode) spawn nanoda subprocess with exported form
5. Compare verdicts. Disagreement â†’ deliberate exit. Crash / timeout in any kernel â†’ AFL catches natively.

## Sequencing
1. Byte-stream RNG refactor of `GenM`. Replace `StdGen` with byte-cursor reader. Smallest change, unblocks everything else.
2. Add lean4lean as Lake dep. In-process second-verdict oracle. Build the differential check on existing `GenM` output first.
3. AFL++ integration on the existing pipeline. `LEAN_CC=afl-clang-fast`, `main` reads stdin, run `afl-fuzz`. Validates the whole driver loop before adding new generators.
4. `GenRawExprM`. Smallest new generator, immediate kernel-crash payoff.
5. `GenExprM` (forward typed composition).
6. Tactic-as-generator path.
7. `GenDeclM` (inductives).
8. nanoda parallel target via cargo-afl.
9. Future: toolchain rebuild with `afl-clang-fast` for C++ kernel + Lean runtime coverage.

## Out of Scope (for now)
- Bidirectional goal-inhabitation generator â€” this is proof search, defer to tactics.
- libFuzzer FFI integration â€” AFL++ via `LEAN_CC` is lower friction and process-based isolation handles Lean's runtime state cleanly. Reconsider only if AFL++ proves limiting.
- Constructing `False`. Differential crashes catch the same bugs (and many more) without needing a full unsoundness construction.
- Shared-memory subprocess coverage merge â€” start with AFL parallel mode, only escalate if needed.
